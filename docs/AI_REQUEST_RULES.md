# AI依頼トリガールール

## 🎯 依頼時の必須トリガー

### 画像生成関連
- `画像生成` / `再生成` → `./run.sh [episode.json] --only [panel_number]`
- `全画像生成` → `./run.sh [episode.json]`
- `明度調整` → 明度安定化プロンプト適用後に生成

### Instagram投稿関連
- `投稿` / `自動投稿` → `./post_instagram.sh [episode_dir]`
- `git pushしてから投稿` → git push後にInstagram投稿
- `キャプション修正` → `instagram_post.json`編集後に投稿

### どんでん返し法則関連
- `どんでん返し` / `法則` → `plot_twist_patterns.json`参照
- `タイトル案` → 仕様準拠の起承転結構成で3案提案
- `はなオチ` → はなが結（オチ）になる構成

## 🔧 よく使う修正パターン

### キャラクター位置修正
```
トリガー: 「位置が違う」「配置を修正」
対応: episode.jsonのdescriptionを修正 → 再生成
```

### セリフ修正
```
トリガー: 「セリフを変更」「吹き出し修正」
対応: episode.jsonのdialogueを修正 → 再生成
```

### フォントサイズ変更
```
トリガー: 「文字が小さい」「フォントサイズ」
対応: module4_image_gen.pyのfont_sizeを修正 → 再生成
```

### 明度調整
```
トリガー: 「明度が違う」「暗い/明るい」
対応: 明度安定化プロンプト適用 → 再生成 → git push → 投稿
```

## 📋 必須確認事項

### 投稿前チェックリスト
- [ ] 最新画像がGitHubに反映されているか（git push済み）
- [ ] キャプションが正しく送信されるか（formatted_caption対応）
- [ ] タイムスタンプ付きURLでキャッシュ回避
- [ ] 明度が安定しているか

### 生成前チェックリスト
- [ ] episode.jsonの内容が最新か
- [ ] キャラクター位置・セリフが正しいか
- [ ] どんでん返し法則に準拠しているか

## 🚨 よくある問題と対策

### GitHubキャッシュ問題
```
症状: 古い画像が投稿される
対策: タイムスタンプ付きURL（?t=1234567890）を自動付与
```

### キャプション送信問題
```
症状: ハッシュタグのみ投稿される
対策: formatted_captionキーを優先的に取得
```

### 明度不安定問題
```
症状: 画像の明度が統一されない
対策: CONSISTENT BRIGHTNESSプロンプトを追加
```

## 💬 効率的な依頼表現

### 良い例
- 「1枚目のキャラクター位置を修正して再生成」
- 「git pushしてからInstagram投稿して」
- 「どんでん返し法則でタイトル案を3つ」

### 避ける表現
- 「適当に調整して」→ 具体的な指示を記載
- 「なんとかして」→ 問題点を明確に特定
- 「雰囲気で」→ 仕様に沿った具体的な指示

## 🔄 標準ワークフロー

1. **画像修正**: episode.json編集 → 再生成
2. **確認**: 画像を視覚的に確認
3. **git push**: 最新画像をGitHubに同期
4. **投稿**: Instagramに自動投稿
5. **検証**: 投稿内容を確認
